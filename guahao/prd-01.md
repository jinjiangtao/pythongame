# CustomTkinter 医院门诊挂号小系统（精简版 V1.0）
基于 Python + CustomTkinter 开发桌面版医院门诊简易挂号系统，全程现代UI组件开发，不使用外部图片、音频资源。代码可直接运行，中文展示正常无乱码，运行流畅无逻辑BUG，支持打包为EXE文件，适合课程实训、新手练手、小型桌面业务系统使用。

## 界面布局
1. 窗口固定尺寸，启动自动居中显示，界面刷新无闪烁，支持浅色/深色主题一键切换
2. 顶部设置系统标题与主题切换按钮，布局简洁规整
3. 主界面分为挂号信息录入区、挂号记录展示区两大板块
4. 底部设置状态栏，展示当前时间与操作提示信息
5. 整体采用圆角、扁平化设计，控件排布均匀，视觉风格简洁清爽

## 核心功能与规则
1. 基础挂号登记：录入患者姓名、性别、就诊科室、接诊医生、联系电话
2. 提交挂号信息，实时同步展示到记录列表
3. 支持一键清空表单内容，重新录入信息
4. 可选中单条记录执行删除、取消挂号操作
5. 基础输入校验，拦截空内容提交等非法操作
6. 数据本地持久化保存，程序重启后数据不丢失
7. 异常场景捕获并弹出中文提示，保障程序稳定运行

## 配套功能
1. 浅色、深色两种主题一键切换，所有组件自动适配配色
2. 各项操作执行后弹出中文提示框，反馈运行状态
3. 挂号列表自动刷新，实时展示最新数据
4. 界面元素样式统一，操作逻辑简单易懂
5. 全代码添加中文注释，组件、事件、逻辑说明完整清晰

## 代码开发规范
1. 采用面向对象方式编码，独立封装主窗口、表单组件、数据展示、基础业务逻辑等模块
2. 界面渲染、事件监听、数据读写、异常处理代码相互解耦
3. 代码命名规范统一，结构层次清晰，便于阅读与二次修改
4. 仅依赖 Python 与 CustomTkinter 库，无额外第三方依赖，兼容性良好
5. 采用 MVC 架构分层设计，视图层、控制层、逻辑层分离，方便后期扩展功能
6. 代码按功能模块拆分，文件不能写到一个文件中，分开写，便于维护。
7. 操作界面采用中文，避免使用英文，确保用户操作方便。
8. 界面好看，符合审美。
