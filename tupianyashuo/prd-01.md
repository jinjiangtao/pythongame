帮我用 Python 和 CustomTkinter 开发一个图片批量压缩工具，界面友好，功能精简实用。代码必须拆分到多个文件，方便维护。

一、文件结构要求

```
image_compressor/
├── main.py           # 程序入口，启动应用
├── app.py            # 主窗口类，负责界面布局和事件绑定
├── compressor.py     # 压缩逻辑类，处理图片压缩算法
├── file_selector.py  # 文件选择相关功能（多选图片、选择输出文件夹）
├── task_manager.py   # 任务管理（列表维护、进度更新、异常处理）
└── utils.py          # 辅助函数（格式化文件大小、弹窗提示等）
```

每个文件职责单一，通过 import 互相调用，不能出现循环导入。

二、功能要求

1. 选择图片：点击按钮选择多张图片（支持 JPG、PNG 格式），选完后在界面上显示文件路径和原始大小。
2. 压缩质量：一个滑动条（50-100），默认 80，实时显示当前数值。
3. 输出文件夹：一个选择文件夹按钮，压缩后的图片保存到这里。
4. 压缩按钮：点击后遍历列表中的每张图片进行压缩，显示进度条。
5. 压缩逻辑：
   · JPG 格式：用 PIL 的 save 方法设置 quality 参数
   · PNG 格式：设置 optimize=True 和 compress_level
   · 压缩后文件名保持原名，不要覆盖原图
6. 结果提示：压缩完成后弹窗提示'完成 X 张，失败 Y 张'。
7. 压缩后的图片可以导出到指定文件夹。

三、界面布局（在 app.py 中实现）

· 窗口大小 600x500，使用 CustomTkinter 深色主题
· 顶部：'选择图片'按钮，旁边显示已选图片数量
· 中间：列表框显示已选图片路径（可滚动）
· 中间偏下：'压缩质量'标签 + 滑动条 + 数值
· 下方：'输出文件夹'按钮 + 显示选中的文件夹路径
· 底部：'开始压缩'按钮 + 进度条

四、技术要求

· 用 customtkinter 做界面
· 用 PIL 处理图片
· 压缩过程不要卡死界面（使用 threading 或 update_idletasks）
· 必须处理异常（损坏图片、格式不支持等），异常信息要友好提示给用户
