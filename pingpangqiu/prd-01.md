使用 Python + Pygame 实现**单机单人版**2D乒乓球对战游戏，**禁止所有代码写在同一文件**，严格模块化分包开发，使用 unittest 编写独立测试文件，结构清晰、低耦合。

特别注意： 
- 所有代码必须在 src 目录下，不能直接在 main.py 中写所有代码。
- 所有测试用例必须在 tests 目录下，不能直接在 main.py 中写所有测试用例。
- 所有代码必须符合 PEP 8 规范，注释规范，可读性高。
- 界面中文字符正常显示，保证游戏界面美观。

目录结构：
- main.py 程序入口与主循环
- src/
  - paddle.py 球拍类（玩家球拍、AI球拍）
  - ball.py 乒乓球类
  - game_core.py 游戏核心逻辑、得分判定
  - config.py 全局配置（窗口大小、速度、颜色、尺寸）
- tests/
  - test_paddle.py
  - test_ball.py

游戏规则（纯单机）：
1. 仅玩家单人操作，对面为AI自动控制；
2. 玩家使用上下方向键控制左侧球拍上下移动；
3. AI自动跟随乒乓球移动，控制右侧球拍接球；
4. 乒乓球碰到上下边界反弹，碰到球拍反弹；
5. 球越过玩家右侧边界 → AI得分；越过AI左侧边界 → 玩家得分；
6. 先拿到设定分数（如10分）的一方获胜，游戏结束；
7. 球拍不能超出游戏窗口上下边界；

要求：类职责单一，测试用例可独立运行，注释规范，可调整难度、速度。
