# Pygame 灯光迷阵 益智游戏 
基于 Python + Pygame 开发单机益智灯光迷阵游戏，纯代码绘制所有界面、图形与交互元素，不依赖外部图片、音频资源。代码可直接运行，中文显示无乱码，运行流畅无卡顿、无逻辑BUG，支持打包为EXE文件，适配课程作业、实训项目使用。

## 一、项目整体说明
灯光迷阵是经典逻辑益智小游戏，核心规则：点击网格内的灯格，会同步翻转自身以及上下左右相邻格子的亮灭状态；玩家目标是将棋盘内所有灯全部点亮即可通关。游戏内置多套难度关卡，搭配计时、步数统计、重置、提示等辅助功能，界面简洁清爽，操作简单、逻辑性强。整体采用面向对象+模块化开发，代码结构清晰，注释完整。

## 二、界面布局设计
1. **主窗口设置**
   固定游戏窗口尺寸，帧率锁定60帧，画面刷新无闪烁、无拖影，窗口标题显示游戏名称。整体配色区分灯亮、灯灭、选中状态，视觉对比明显。

2. **主菜单界面**
   - 顶部居中展示游戏大标题；
   - 中部排列功能按钮：开始游戏、关卡选择、游戏规则、退出游戏；
   - 底部添加简短游戏简介文字；
   - 按钮支持鼠标悬浮高亮、点击反馈效果。

3. **关卡选择界面**
   以网格形式展示所有解锁关卡，区分已通关、未通关、锁定关卡状态；点击对应关卡即可进入对局；配备返回菜单按钮。

4. **游戏主界面**
   - 中心区域：核心游戏棋盘，由等大正方形灯格组成规整网格；
   - 顶部状态栏：实时显示当前关卡、已走步数、本局用时；
   - 右侧/底部功能区：放置重新开始、本局提示、返回菜单三个功能按钮；
   - 所有UI元素布局匀称，间距统一，适配窗口大小。

5. **通关弹窗界面**
   本局全部灯点亮后自动弹出胜利弹窗，展示当前关卡、总步数、总用时；提供「下一关」「重新挑战」「返回菜单」三个操作选项。

## 三、游戏核心规则与功能
### （一）基础游戏规则
1. 棋盘由 N×N 矩阵灯格构成，每一格存在**点亮**、**熄灭**两种状态。
2. 鼠标点击任意一个灯格，触发翻转逻辑：当前格子 + 其上、下、左、右四个相邻格子，统一翻转亮灭状态（亮变灭、灭变亮）；超出棋盘边界的位置不做处理。
3. 禁止无效操作判定：仅响应棋盘内格子的点击，点击空白区域无任何反应。
4. 胜利条件：棋盘内**所有灯格全部处于点亮状态**，立即判定本局通关。

### （二）难度与关卡系统
1. 关卡梯度设计，尺寸逐步增大、初始布局复杂度提升：
   - 入门关卡：3×3 网格，布局简单，新手上手；
   - 普通关卡：4×4 网格，难度适中；
   - 进阶关卡：5×5 / 6×6 网格，格子数量多，解谜难度高。
2. 关卡解锁机制：通关当前关卡后，自动解锁下一关；未解锁关卡无法点击进入。
3. 支持自由选择已解锁关卡重复游玩。

### （三）数据统计功能
1. 步数统计：每有效点击一次灯格，步数计数+1，通关后保留本局步数。
2. 计时功能：进入游戏对局后自动开始计时，精确到秒，格式为 `分:秒`；通关后停止计时。

### （四）辅助功能
1. 重新开始：点击后重置当前关卡布局，步数、计时清零，回到本局初始状态。
2. 提示功能：每局限定使用次数（如3次），点击后自动高亮一个最优点击位置，辅助玩家解谜；次数用完后提示按钮置灰不可点击。
3. 游戏规则查看：菜单内可打开规则说明弹窗，图文结合讲解玩法与操作。

## 四、交互体验优化
1. 鼠标悬浮在灯格上时，格子边框变色高亮，给予视觉提示。
2. 点击灯格时有简短的视觉反馈，翻转状态过渡自然。
3. 按钮悬浮、点击均有状态变化，区分可点击/不可点击状态。
4. 切换界面、弹出弹窗过程流畅，无界面卡顿、层级错乱问题。
5. 所有中文文字正常渲染，无乱码、无字体偏移。

## 五、代码架构与开发规范
1. **架构设计**
   采用面向对象思想拆分模块，独立封装各类功能类：
   - 界面管理类：负责菜单、弹窗、页面切换；
   - 棋盘灯格类：单个格子状态、绘制、坐标管理；
   - 游戏逻辑类：状态翻转、通关判断、关卡数据管理；
   - 统计工具类：步数计数、计时器功能。
   各模块职责分离，低耦合、高复用。
模块分开编写，模块分开编写，不能写到一个文件里。 方便后面开发和维护。


2. **逻辑封装要求**
   - 灯格状态翻转、相邻格子判定、通关检测核心算法单独封装函数；
   - 所有关卡初始布局使用二维数组统一配置，方便新增、修改关卡。

3. **注释规范**
   所有类、函数、核心变量、关键算法、界面元素、分支逻辑均添加**详细中文注释**，代码可读性强，便于二次修改和功能扩展。

4. **环境与兼容性**
   仅依赖 Pygame 官方库，不引入任何第三方拓展库；代码兼容 Windows 系统，可正常打包为独立EXE文件，脱离Python运行环境也能使用。

5. **异常处理**
   增加基础边界判断，防止数组越界、重复触发逻辑等问题，保证程序全程稳定不崩溃。
